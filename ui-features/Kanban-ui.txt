## 📏 Recomendação prática (usando Tailwind)

### ✅ **Largura total do Kanban**

Use **máximo de 1440px** em monitores grandes e **margens automáticas** para centralizar:

```html
<div class="mx-auto max-w-[1440px] px-8 md:px-12 lg:px-16 xl:px-20">
  <!-- Kanban aqui -->
</div>
```

**Explicação:**

- `mx-auto` → centraliza horizontalmente.
- `max-w-[1440px]` → limita a largura total.
- `px-8 md:px-12 lg:px-16 xl:px-20` → cria **Sidemargins adaptativas**:
    - `px-8` ≈ 32px (mobile)
    - `px-12` ≈ 48px (tablet)
    - `px-16` ≈ 64px (desktop)
    - `px-20` ≈ 80px (telas grandes)

Essas medidas equivalem a **margens laterais de 5–7%** do viewport, exatamente como na sua imagem.

---

## 💡 Alternativa proporcional (100% da largura)

Se você quer **escala fluida** sem `max-width`, use:

```html
<section class="px-[5vw] lg:px-[7vw]">
  <!-- Kanban -->
</section>

```

👉 Isso mantém o Kanban sempre com **5–7% de respiro em cada lado**, proporcional ao tamanho da tela.

## 🧱 Estrutura base recomendada (para o layout do Kanban)

```html
<main class="min-h-screen bg-cinza-claro">
  <header class="py-6 px-[5vw] border-b border-cinza-200">

  </header>

  <section class="py-10 px-[5vw]">
    <div class="rounded-xl bg-white shadow-md p-6">
      <!-- Componente do Kanban aqui -->
    </div>
  </section>
</main>

```

    - Margens laterais → ~5–7% da tela cada.

    - Layout adaptativo e limpo para telas de qualquer tamanho.

# Estrutura UI das Colunas do Kanban:

## 🎨 1. Estrutura Hierárquica

```
📦 KanbanColumn
 ├── 🧭 Header da Coluna
 │     ├── Ícone de Status (dot color)
 │     ├── Nome da Coluna (Título)
 │     └── Contador de Fichas (badge)
 │
 ├── 🗂️ Corpo da Coluna
 │     ├── Área de Drop (drag zone)
 │     └── Mensagem de Estado (ex: Nenhuma ficha / Arraste aqui)
 │
 └── 🪶 Sombra + Espaçamento + Radius
```

## 🎯 2. Tokens Utilizados

| Elemento | Token | Valor / Origem | Descrição |
| --- | --- | --- | --- |
| **Fundo da coluna** | `--branco` | `global.css` | Fundo base das colunas. |
| **Borda externa** | `border-gray-200` | Tailwind default / DS neutral | Discreta e limpa. |
| **Sombra** | `--shadow-sm` | `0 1px 2px rgba(0,0,0,.06)` | Profundidade leve. |
| **Radius** | `--radius-xl` | `20px` | Arredondamento elegante. |
| **Padding interno** | `--sp-6` | `24px` | Espaçamento interno padrão. |
| **Cor do título** | `--preto` | `global.css` | Título sempre legível. |
| **Cor do placeholder** | `--cinza` | `global.css` | Textos secundários. |
| **Cor dos dots (status)** | `blue`, `green`, `yellow`, `red` | Token custom `--status-{color}` | Ícone visual da coluna. |

## 🧭 3. Header da Coluna:

**Composição:**

- Fundo branco com leve sombra.
- Padding vertical 12px / horizontal 16px.
- Flex horizontal com gap 8px.
- Border-bottom cinza clara (#e5e7eb).
- Ícone circular colorido representando o status da coluna.
- Título (H4 → `.as-t4`).
- Contador de fichas (badge minimalista).

**Props:**

| Prop | Tipo | Exemplo | Descrição |

| `title` | string | `"Feitas / Cadastrar no MK"` | Nome da coluna. |
| `count` | number | `0` | Quantidade de fichas. |

## 🗂️ 4. Corpo da Coluna

**Composição:**

- Fundo branco.
- Altura mínima: `min-h-[400px]`.
- Conteúdo centralizado (flex + justify-center + items-center).
- Mensagem com ícone central (gray 300).
- Texto principal (`Nenhuma ficha`) → `.as-t4`.
- Subtexto (`Arraste fichas aqui`) → `.as-t8`.

**Estado vazio:**

```

Arraste fichas aqui

```

**Estado com cards:**

- Cards listados verticalmente com espaçamento entre eles (`gap-3`).
- Cada card herda a UI `.card-select`.

# Estrutura UI dos Cards nas colunas:

## 1) Estrutura visual

```
📦 Card (container)
 ├─ Linha 1: Public.applicantes.primary_name + Menu (⋮) 
 ├─ Linha 2: CPF ou CNPJ (Conforme public.Applicants.person_type): public.applicants.cpf_cnpj 
 ├─ Linha 3: Icone de Telefone: Public.applicantes.phone  | Icone de Whatsapp: Public.applicantes.whatsapp 
 ├─ Linha 4: Data de Agendamento: public.kanban_cards.due_at | Horário: public.kanban_cards.hora_at
 └─ Linha 5: Bairro: Public.applicantes.bairro
```

**Layout geral**

- Fundo: `base.white`
- Borda: 1px `#e5e7eb`
- Radius: `-radius-xl` (20px)
- Sombra: `-shadow-sm` (elevado no hover para `-shadow-md`)
- Padding: 16px interno
- Gap vertical entre linhas: 6–8px
- Linha divisória opcional (hairline) antes do rodapé/“chips”, se houver

## 2) Tipografia (usa seus “T-tokens”)

- **Nome (Linha 1):** `.as-t4` **+** `font-semibold` (cor `brand.secondary` `#000`)
- **Demais rótulos/valores:** `.as-t8` para rótulo, `.as-t4` light/regular para valor
    
    (cores neutras: `#6b7280` para rótulo, `#111827` para valores)
    
- Espaçamento de linhas: use line-height dos tokens (`-t4-line`, `-t8-line`)

---

## 3) Cores & ícones

- Ícones (telefone, whatsapp, calendário, relógio, mapa): `#6b7280`
- Links/ações (telefone/whats clicáveis): hover em `brand.primary` cor:`Primária`
- Menu (⋮): ícone `#9ca3af`, hover `#6b7280`

---

## 4) Estados e interações

- **Hover do Card:** `shadow-md`, translateY(-2px), borda mantém `#e5e7eb`
- **Focus (via teclado):** outline com halo `brand.primary` (0 0 0 3px)
- **Dragging (arrastando):** opacidade 0.92 + `shadow-lg`

---

## 5) Variantes (opcionais)

- **Sem algum campo:** esconde Linha, responsiva outras linhas para o lugar da linha escondido, priorizando o UX

---

## 6) Acessibilidade

- Botão de menu (⋮) com `aria-label="Ações da ficha: {Nome}"`
- Links de telefone/WhatsApp com `aria-label` claro
- Títulos com `<h3>` semântico dentro do card

---

## 7) Grid/Medidas

- Largura da coluna define o card.
- Largura mínima do card: `min-w-[280px]` para mobile.
- Espaço entre cards: `gap-3`/`gap-4`.
